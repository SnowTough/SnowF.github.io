tk.mybatis的使用
2023-02-10|MyBatis

Tk.mybatis 对底层的 sql 进行抽象封装，基本不需要再在 Mapper 中写 sql 语句了，只需要按照逻辑思维，遵循 Tk.mybatis 语法来实现 Service

添加依赖

使用方法

  1. 创建 Dao 层的 Mapper 接口，继承 tk.mybatis.mapper.common.Mapper 接口，Mapper 接口要指明数据的对象类
  1. 在 Application 启动类使用 @MapperScan 标明 Mapper 扫描的路径
  1. 所以，tk.mybatis 主要使用于 service 层的接口实现类中（xxxServiceImpl.class)

删除

​ DeleteByPrimaryKeyMapper 接口有一个方法名为 deleteByPrimaryKey，方法通过找到对应的 Java 实体类的主键作为判断条件，对数据进行删除操作

​ DeleteMapper 接口有一个方法名为 delete，该方法的参数 Object var1 就是数据库表对应的 Java 实体类，参数实体中的哪些字段不为 null，就会被作为删除 sql 语句的条件字段，条件关系为 and（&&），而不是 or（ || ）

插入

​ InsertMapper 类有一个方法名为 insert，该方法会往数据表中插入一条记录，记录的字段数等于实体类的字段数

​ InsertSelective 类有一个方法名为 insertSelective，该方法内实体类参数为 null 的字段会被忽略，在 tkmybatis 中生成的 insert sql 语句只会包含不为 null 的字段
